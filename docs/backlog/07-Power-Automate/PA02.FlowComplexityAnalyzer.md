## Notes
- Core data: `workflow` rows (`category = 5`), definition JSON in `clientdata`. Everything scored here is static structure — no run metrics required, so it is fully SDK-feasible.
- Scoring engine consumes the parsed action tree (actions, conditions, switches, scopes, loops/`foreach`/`until`, parallel branches, child flows, connectors, expressions, `runAfter` config, retry policies) and emits a 0–100 score plus a band.
- Bands: Simple / Moderate / Complex / High Risk / Refactor Recommended. Keep the rule weights and thresholds in a UI-free, unit-tested scoring class (like the shipped analyzers) so they can be tested without Dataverse.
- Reuse DRA's `clientdata` parser and, ideally, PA01's dependency model — one parser, two consumers.
- Read-only; output is score cards, an action-count summary, a findings grid, and refactoring recommendations. No writes.
- Redact any secrets/URLs encountered while parsing; the tool reports structure, not payloads.

## EPIC-PA02 — Score cloud-flow complexity and drive refactoring
> **As** a **MGR** (delivery/architecture lead), **I want** an objective 0–100 complexity score and refactoring guidance per flow, **so that** I can target the flows that are hardest to maintain and riskiest to support.

**Outcome:** a per-flow complexity score with band, an action/nesting breakdown, prioritized maintainability findings, and an exportable complexity report.

---

## FEAT-PA02-1 — Flow selection and definition parsing `[Planned]`
- **US-PA02.1.1** `[Planned]` **As** an architect, **I want** flows listed by solution/environment and their definition JSON parsed, **so that** I have the structure to score.
  - **AC:** Flows load via `RetrieveAll` off the UI thread with progress/cancel; `clientdata` is parsed into an action tree, and parse failures degrade to an Info finding.
- **US-PA02.1.2** `[Planned]` **As** an architect, **I want** to select one, many, or all flows to analyze, **so that** I can score a single flow or a whole solution.
  - **AC:** Multi-select drives a batch scoring run with per-flow progress.

## FEAT-PA02-2 — Structural counting and nesting analysis `[Planned]`
- **US-PA02.2.1** `[Planned]` **As** an architect, **I want** counts of actions, conditions, switches, scopes, loops, parallel branches, child flows, and connectors, **so that** I can see a flow's size at a glance.
  - **AC:** An action-count summary shows every category; counts feed the score.
- **US-PA02.2.2** `[Planned]` **As** an architect, **I want** deeply nested logic and excessive custom expressions detected, **so that** I can spot hard-to-read flows.
  - **AC:** Maximum nesting depth and expression count are computed; values over configurable thresholds raise findings.

## FEAT-PA02-3 — Risk and hygiene detection `[Planned]`
- **US-PA02.3.1** `[Planned]` **As** a support lead, **I want** missing error-handling scopes, missing retry policies, and missing run-after-failure handling detected, **so that** I know which flows fail silently.
  - **AC:** Each hygiene gap is a distinct finding with severity; run-after and retry config are read from `clientdata`.
- **US-PA02.3.2** `[Planned]` **As** an architect, **I want** too many Dataverse operations, too many HTTP/custom-connector actions, and hardcoded values detected, **so that** I can spot cost and portability risks.
  - **AC:** Threshold-based findings for Dataverse/HTTP action volume and hardcoded literals (redacting secrets).

## FEAT-PA02-4 — Scoring, banding, and recommendations `[Planned]`
- **US-PA02.4.1** `[Planned]` **As** a MGR, **I want** a 0–100 complexity score and a band (Simple → Refactor Recommended) per flow, **so that** I can rank and prioritize.
  - **AC:** The UI-free scoring class combines the weighted factors (action count, nesting, connectors, Dataverse/HTTP actions, conditions/loops, error-handling gaps, hardcoded values, child-flow deps); score and band show on a score card and are unit-tested.
- **US-PA02.4.2** `[Planned]` **As** an architect, **I want** specific refactoring recommendations per finding, **so that** I know how to reduce the score.
  - **AC:** Recommendations map to the findings that drove the score (e.g., "wrap in a Scope with configured run-after", "replace 6 sequential Dataverse calls with a batch pattern").

## FEAT-PA02-5 — Dashboard and export `[Planned]`
- **US-PA02.5.1** `[Planned]` **As** a MGR, **I want** a complexity dashboard across the selected scope, **so that** I can see the distribution of scores and the worst offenders.
  - **AC:** Dashboard shows score distribution and a sortable grid; clicking a flow opens its definition tree and findings.
- **US-PA02.5.2** `[Planned]` **As** an architect, **I want** the complexity report exported to Excel/PDF/JSON/HTML, **so that** I can share it or track it over time.
  - **AC:** Export runs off the UI thread; JSON carries per-flow scores, bands, factor breakdown, and findings.

- Export formats: Excel, PDF, JSON, HTML.
