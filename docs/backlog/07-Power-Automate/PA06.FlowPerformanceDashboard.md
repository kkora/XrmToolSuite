> **Overlaps:** PA02 Flow Complexity Analyzer (static complexity/error-handling factors), PA03 Flow Failure Investigator (failure signals), and Deployment Risk Analyzer. This dashboard aggregates those static signals into KPIs; its *runtime* metrics half is where the feasibility caveat bites.

## Notes
- **FEASIBILITY CAVEAT (read first):** Flow **run metrics — average/maximum duration, failure rate, retry counts, run frequency, high-volume detection — are NOT available through the Dataverse SDK / `IOrganizationService`.** They live in the Power Automate / Logic Apps runtime (Flow Management / analytics APIs, separate auth from the org connection). The source hedges every metric with "where available". **Do not promise a live runtime dashboard from the org connection.** Design so the dashboard is fully useful from *static* signals, and treats runtime metrics as an optional, clearly-gated enhancement (separate auth/endpoint) that degrades to "not available" rather than showing blanks or zeros.
- Feasible-from-SDK signals (the always-on half): flow inventory and active/inactive state (`workflow`), connector usage, complexity (reuse PA02), missing error-handling scopes, sequential vs. parallel operations, excessive Dataverse/API call counts, many loops/branches — all from `clientdata`.
- Runtime-dependent signals (the gated half): recent run metrics, average/max duration, failure rate, retry patterns, frequent-failure and long-running detection — only if a Power Automate/analytics API connection is wired.
- Performance scoring engine (UI-free, unit-tested) blends available factors (runtime duration, failure rate, retry behavior, connector risk, complexity, error-handling quality, Dataverse operation volume, external-API dependency) — weighting gracefully around missing runtime inputs.
- Baseline/trend: persist a saved baseline as a serializable POCO setting (no credentials) and compare later runs. When runtime metrics are unavailable, baseline compares the static factors only.
- Reuse PA02's complexity model and DRA/PA01 `clientdata` parser. Read-only; charts + export only. Never expose HTTP trigger URLs or secrets.

## EPIC-PA06 — Surface slow, unreliable, and risky flows at a glance
> **As** a **MGR**, **I want** a dashboard ranking flows by performance and runtime risk, **so that** I can act on problem flows before they cause business-process failures.

**Outcome:** KPI cards and charts over the flow estate, a per-flow performance score, a baseline comparison, and an export — static signals always, runtime metrics where available.

---

## FEAT-PA06-1 — Flow inventory and static risk signals `[Planned]`
- **US-PA06.1.1** `[Planned]` **As** a MGR, **I want** flow inventory with active/inactive status and connector usage, **so that** I see the estate at a glance.
  - **AC:** Inventory loads via `RetrieveAll` off the UI thread with progress/cancel; active/inactive and connector usage render on KPI cards and a grid.
- **US-PA06.1.2** `[Planned]` **As** a MGR, **I want** static performance-risk signals surfaced (missing error handling, excessive Dataverse/API calls, many loops/branches, sequential operations that could be parallelized, high complexity), **so that** I can spot risky flows without run data.
  - **AC:** Each static signal is computed from `clientdata` (reusing PA02 complexity model) and shown as a finding/flag per flow.

## FEAT-PA06-2 — Runtime metrics (optional, gated) `[Planned]`
- **US-PA06.2.1** `[Planned]` **As** a MGR, **I want** the dashboard to state clearly when runtime metrics are unavailable through the current connection, **so that** I don't mistake "no data" for "healthy".
  - **AC:** When no analytics/Power Automate API is configured, duration/failure/retry panels show an explicit "runtime metrics not available via Dataverse" state — never blank cards or fabricated zeros.
- **US-PA06.2.2** `[Planned]` **As** a MGR, **I want** recent run metrics — average/max duration, failure rate, retry patterns, high-volume and long-running flows — *where the Power Automate API is available and authorized*, **so that** I can see real runtime behavior.
  - **AC:** If (and only if) an optional analytics connection is provided, metrics load off the UI thread and feed the duration/failure/connector charts; the separate auth dependency is documented.

## FEAT-PA06-3 — Performance scoring `[Planned]`
- **US-PA06.3.1** `[Planned]` **As** a MGR, **I want** a per-flow performance score across the standard categories (duration, failure rate, retry, connector risk, complexity, error-handling quality, Dataverse volume, external-API dependency), **so that** I can rank flows.
  - **AC:** The UI-free scoring engine blends available factors, weighting around missing runtime inputs, and marks whether a score is static-only or runtime-augmented; unit-tested.
- **US-PA06.3.2** `[Planned]` **As** a MGR, **I want** performance findings with recommendations, **so that** I know how to improve a flagged flow.
  - **AC:** Findings panel maps each risk factor to a recommendation (e.g., "parallelize independent Dataverse calls", "add retry policy").

## FEAT-PA06-4 — Baseline, trends, and export `[Planned]`
- **US-PA06.4.1** `[Planned]` **As** a MGR, **I want** to save a baseline and compare later runs, **so that** I can see whether the estate is improving.
  - **AC:** Baseline persists as a serializable POCO (no credentials) via Load/SaveSettings; a comparison tab shows deltas; when runtime metrics are absent, comparison covers static factors only.
- **US-PA06.4.2** `[Planned]` **As** a MGR, **I want** the dashboard exported to Excel/PDF/JSON/HTML, **so that** I can report to stakeholders.
  - **AC:** Export runs off the UI thread; the report flags which metrics are static vs. runtime-derived ("where available"); secrets/URLs redacted.

## Definition of Done
- Follows suite conventions (BaseToolControl, RunAsync/RetrieveAll, Load/SaveSettings, progress+cancel).
- Read-only default; scoring engine is UI-free and unit-tested; reuses PA02 complexity model and shared `clientdata` parser; runtime-metrics feasibility caveat honored — no runtime claim unless the Power Automate/analytics API is configured; secrets and HTTP trigger URLs never exposed.
- Export formats: Excel, PDF, JSON, HTML.
- Testing skeleton under testing/FlowPerformanceDashboard/ when implementation starts.
