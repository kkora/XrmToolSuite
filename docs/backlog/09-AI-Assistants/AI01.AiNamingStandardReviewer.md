## Notes
- Deterministic rules first: casing/prefix/length/abbreviation/generic-word/duplicate checks over `RetrieveAllEntitiesRequest` / `RetrieveEntityRequest` metadata (`EntityMetadata`, `AttributeMetadata`, `RelationshipMetadata`), plus web resources, plugin assemblies/types/steps, custom APIs, environment variables, connection references, business rules and workflows. All findings available with AI disabled.
- Publisher-prefix rules resolve against the solution's publisher; "wrong prefix" = component prefix ≠ owning solution's publisher prefix.
- AI is opt-in and clearly labelled advisory: it only *suggests* better display names / schema names / descriptions and explains why a name is weak — it never mutates metadata.
- Reuse AI Solution Reviewer's AI client: session-only API key (never persisted), consent preview of the exact redacted payload before any send, GUIDs/URLs/user names/emails masked, audit trail of AI outputs, offline fallback to deterministic report on AI failure.
- Prompts/system-instructions are configurable templates surfaced in settings.
- Read-only tool. Metadata reads via `Service.RetrieveAll` / metadata requests off the UI thread through `RunAsync`; progress + cancellation; settings round-trip on Load/ClosingPlugin. Keep the naming-rules engine UI-free so it is unit-testable and CI-liftable.

## EPIC-AI01 — Score and improve naming consistency across a solution or environment
> **As** an ARCH / TOOLDEV, **I want** deterministic naming findings with an optional AI layer that proposes better names and descriptions, **so that** I can drive naming standards without hand-auditing every component.

**Outcome:** a naming quality score, a per-component findings grid classified by rule, and (opt-in) AI-suggested names/descriptions — with the tool fully useful when AI is off.

---

## FEAT-AI01-1 — Deterministic rules-first findings `[Planned]`
- **US-AI01.1.1** `[Planned]` **As** a TOOLDEV, **I want** the full naming review to run without any AI, **so that** the tool is useful and safe with AI disabled.
  - **AC:** Complete findings and score available offline; AI toggled off by default.
- **US-AI01.1.2** `[Planned]` **As** an ARCH, **I want** to scan a selected solution or the whole environment across tables, columns, relationships, forms, views, charts, dashboards, web resources, plugin assemblies/types/steps, custom APIs, environment variables, connection references, business rules and workflows, **so that** no component type is a naming blind spot.
  - **AC:** Component reads via `RetrieveAll`/metadata requests off the UI thread with progress and cancellation.
- **US-AI01.1.3** `[Planned]` **As** an ARCH, **I want** rule checks for missing/wrong publisher prefix, inconsistent casing, abbreviations, unclear names, duplicates, too-long/too-short names, generic names (test/temp/old/new/copy/backup) and environment-specific names, **so that** every violation class is caught deterministically.
  - **AC:** Each finding carries rule id, severity (Critical/High/Medium/Low/Info) and the offending value.

## FEAT-AI01-2 — Naming quality score & dashboard `[Planned]`
- **US-AI01.2.1** `[Planned]` **As** an MGR, **I want** a naming quality score with per-category breakdown, **so that** I can track naming health over time.
  - **AC:** Score is deterministic and reproducible with AI off; category cards drill into the findings grid.
- **US-AI01.2.2** `[Planned]` **As** an ARCH, **I want** violations grouped by rule and component type, **so that** I can prioritise systemic issues over one-offs.

## FEAT-AI01-3 — AI naming suggestions (opt-in) `[Planned]`
- **US-AI01.3.1** `[Planned]` **As** an ARCH, **I want** AI-suggested display names, schema names and descriptions with an explanation of why the current name is weak, **so that** I get concrete rename candidates.
  - **AC:** Consent preview of the exact redacted payload before send; session-only key; GUIDs/URLs/emails/names masked; output labelled advisory; audit trail retained; offline fallback on failure.
- **US-AI01.3.2** `[Planned]` **As** a TOOLDEV, **I want** AI to group similar naming violations into themes, **so that** I can fix a class of names in one pass.
  - **AC:** Grouping is advisory only; never mutates metadata.

## FEAT-AI01-4 — Naming convention guide generation (opt-in) `[Planned]`
- **US-AI01.4.1** `[Planned]` **As** an MGR, **I want** AI to generate a naming convention guide from the findings and my standard settings, **so that** the team has a written standard to adopt.
  - **AC:** Generated guide is clearly marked AI-drafted; deterministic standard settings are the source of truth; export available with AI off (empty AI section).

## FEAT-AI01-5 — Configurable standards & prompt templates `[Planned]`
- **US-AI01.5.1** `[Planned]` **As** a TOOLDEV, **I want** editable naming-standard settings and AI prompt/system-instruction templates, **so that** the review matches our house style.
  - **AC:** Settings and templates round-trip via Load/ClosingPlugin as plain POCOs; no key or connection details persisted.

## FEAT-AI01-6 — Export `[Planned]`
- **US-AI01.6.1** `[Planned]` **As** an ARCH, **I want** to export the naming review (deterministic findings plus any AI suggestions) to Excel/PDF/Word/CSV/Markdown/HTML/JSON, **so that** I can circulate it.
  - **AC:** Export runs off the UI thread with progress and cancellation; sensitive values masked in output; AI sections labelled advisory.
