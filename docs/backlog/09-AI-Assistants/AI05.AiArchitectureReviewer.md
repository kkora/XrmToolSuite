## Notes
- Deterministic first: analyze solution structure, table design, relationships, forms/views/apps, plugins/custom APIs, flows, JavaScript/business rules, environment variables/connection references, security roles, solution layering, and documentation completeness — reusing Solution Complexity Score / Knowledge Graph / Technical Debt signals. Detectors: architecture anti-patterns, duplicated business logic across layers, over-customized tables, poor separation of concerns, hardcoded environment values.
- Produce a deterministic architecture score and findings grid with severity — fully available with AI off.
- AI is opt-in and clearly labelled advisory: write the architecture review, classify patterns/anti-patterns, identify modernization opportunities, propose a target architecture and ADR suggestions — it never changes any component.
- Reuse AI Solution Reviewer's AI client: session-only API key (never persisted), consent preview of the exact redacted payload before any send, GUIDs/URLs/user names/emails masked, audit trail, offline fallback to deterministic report on failure. Prompts/system-instructions configurable.
- Read-only tool. Reads via `Service.RetrieveAll` off the UI thread through `RunAsync`; progress + cancellation; settings round-trip. Keep the architecture-rules aggregation UI-free and unit-testable.

## EPIC-AI05 — Produce a structured architecture review and modernization roadmap
> **As** an ARCH, **I want** deterministic architecture findings with an optional AI layer that writes the review and proposes a target architecture, **so that** I can assess and communicate design quality without a manual deep-dive.

**Outcome:** an architecture score, anti-pattern/design findings, and (opt-in) an AI architecture review, modernization roadmap and ADR suggestions — deterministic findings stand alone with AI off.
