## Notes
- Deterministic first: consume/execute the SEC-track analyzers for roles, role privileges/scope, users, teams, inactive-but-privileged users, admin-equivalent users, field security profiles, sharing/PrincipalObjectAccess risks, audit settings, sensitive columns, and connection-reference owners. Detectors: over-privileged users, broad global privileges, risky delete/assign/share privileges, sensitive fields without field-security/audit.
- Produce a deterministic security score and findings grid with severity (Critical/High/Medium/Low/Info) — fully available with AI off.
- **Redaction is paramount for this tool:** never send user names, emails, GUIDs, URLs, or actual sensitive-column values to AI. AI receives only masked/aggregated findings.
- AI is opt-in and clearly labelled advisory: summarise posture, explain risk in plain language, generate remediation plan and role-rationalisation guidance — it never edits roles, users, or settings.
- Reuse AI Solution Reviewer's AI client: session-only API key (never persisted), consent preview of the exact redacted payload before any send, audit trail of AI outputs, offline fallback to the deterministic report on failure. Prompts/system-instructions configurable.
- Read-only tool. Reads via `Service.RetrieveAll` off the UI thread through `RunAsync`; progress + cancellation; settings round-trip. Keep the security-rules aggregation UI-free and unit-testable.

## EPIC-AI03 — Explain Dataverse security posture and remediation to admins, auditors and executives
> **As** a SEC lead / MGR, **I want** deterministic security findings with an optional AI layer that explains risk in plain language and drafts a remediation plan, **so that** I can act on and communicate posture without deep manual analysis.

**Outcome:** a security score, findings across roles/users/teams/fields/sharing/audit, and (opt-in) AI executive and auditor summaries plus a remediation checklist — with deterministic findings standing alone when AI is off.

---

## FEAT-AI03-1 — Deterministic rules-first findings `[Planned]`
- **US-AI03.1.1** `[Planned]` **As** a SEC lead, **I want** the full security review and score without any AI, **so that** the tool is useful and safe with AI disabled.
  - **AC:** Complete deterministic findings and score available offline; AI toggled off by default.
- **US-AI03.1.2** `[Planned]` **As** a SEC lead, **I want** detection of over-privileged users, broad global privileges, risky delete/assign/share privileges, inactive-but-privileged users, admin-equivalent users and sensitive fields lacking field-security/audit, **so that** the real risks surface deterministically.
  - **AC:** Reads via `RetrieveAll` off the UI thread with progress and cancellation; each finding carries severity and the concrete evidence.

## FEAT-AI03-2 — Security dashboard & scoring `[Planned]`
- **US-AI03.2.1** `[Planned]` **As** an MGR, **I want** a security score with risk-category cards and a user/role/team findings grid, **so that** I can see risk concentration at a glance.
  - **AC:** Score deterministic and reproducible with AI off.
- **US-AI03.2.2** `[Planned]` **As** a SEC lead, **I want** a dedicated sensitive-data panel, **so that** columns needing protection are obvious.
  - **AC:** Sensitive values are shown masked and never leave the tool unmasked.
