## Notes
- **Do not rebuild the debt engine.** Consume the existing Technical Debt Analyzer's deterministic findings (unused tables/columns, duplicate metadata, deprecated JS APIs, inactive rules/workflows, duplicate plugin steps, unmanaged prod customizations, missing descriptions, stale components, unused environment variables/web resources, over-complex forms/views) and its debt score.
- AI is opt-in and clearly labelled advisory: it summarises, groups by theme, prioritises remediation, and estimates effort/value — it never changes anything in the environment.
- Reuse AI Solution Reviewer's AI client: session-only API key (never persisted), consent preview of the exact redacted payload before any send, GUIDs/URLs/user names/emails masked, audit trail of AI outputs, offline fallback to the deterministic report on AI failure.
- Prompts/system-instructions are configurable templates in settings.
- Read-only tool. Findings come from metadata/queries via `Service.RetrieveAll` off the UI thread through `RunAsync`; progress + cancellation; settings round-trip. Keep the debt-consumption/aggregation logic UI-free and unit-testable.

## EPIC-AI02 — Turn deterministic technical-debt findings into a prioritised, business-ready backlog
> **As** an MGR / ARCH, **I want** an AI advisory layer over deterministic debt findings that summarises impact and prioritises cleanup, **so that** I can plan remediation without translating raw findings by hand.

**Outcome:** an executive summary, an effort/value matrix, and a prioritised remediation backlog with acceptance criteria — all backed by deterministic findings that stand alone when AI is off.
